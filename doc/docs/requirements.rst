Cross Industry RCM Requirements
================================



These tables contain the requirements for data interchange functionality identified during the :ref:`IMPRCM project <intro-imprcm>`. During this work, these were developed based on existing requirements from: 


- T1010-01 data architecture (see :ref:`intro-t1010`)
- Work so far on the :term:`In2Rail` data / application architecture.

These sources were reviewed and assessed to create a fuller and more up to date set of requirements, with some :ref:`additional requirements <imprcm-requirements>` created to fill gaps, and some that were :ref:`duplicated <tab_requirements_remove>` removed.  

The new set of Cross Industry RCM requirements is shown below, categorised in tables as follows:

- :ref:`Broker Operation <tab_requirements_brokerOperation>`
- :ref:`Unique ID <tab_requirements_uniqueID>`
- :ref:`Asset ID <tab_requirements_assetId>`
- :ref:`Event Data <tab_requirements_eventData>`
- :ref:`Location <tab_requirements_location>`
- :ref:`Time <tab_requirements_time>`
- :ref:`Metadata <tab_requirements_metadata>`
- :ref:`Reference Data <tab_requirements_refdata>`
- :ref:`Interchange Standards <tab_requirements_interchangeStandards>`
- :ref:`Schema <tab_requirements_schema>`
- :ref:`REST API <tab_requirements_rest>`
- :ref:`Service Oriented <tab_requirements_serviceOriented>`
- :ref:`Support <tab_requirements_support>`
- :ref:`Security <tab_requirements_security>`
- :ref:`Logging <tab_requirements_logging>`
- :ref:`Rolling Stock <tab_requirements_rollingStock>`
- :ref:`Ontologies <tab_requirements_ontologies>`


As part of the IMPRCM project, a set of :ref:`requirements-ugms` for a data broker to deliver the UGMS case study was also developed.
